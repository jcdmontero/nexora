# Plan: PWA Fase 6 — Offline Avanzado para NEXORA

## Objetivo
Permitir que usuarios en bodegas/talleres con internet inestable puedan seguir operando cuando se cae la conexión. Las operaciones críticas se encolan localmente y se sincronizan cuando vuelve internet.

## Contexto
- **Stack actual**: Laravel 13 + Inertia.js + React 19
- **Service Worker**: ya existe en `public/sw.js` (cache first para assets, network first para páginas)
- **Sin soporte offline actual**: no hay detección de red, IndexedDB, ni cola de sincronización
- **Dato clave**: Inertia usa requests HTML (no JSON API). Para offline necesitamos endpoints API ligeros para las operaciones críticas.

## Alcance (priorizado por impacto de negocio)

### Operaciones críticas offline:
1. **POS** — Crear venta sin internet (taller/bodega)
2. **Service Desk** — Crear orden de servicio (recepción de equipo)
3. **Dashboard** — Ver datos cacheados al abrir la app sin conexión

## Archivos a crear/modificar

### 1. `resources/js/lib/offline-db.ts` (CREAR)
Wrapper ligero sobre IndexedDB sin dependencias externas:
- `openDB()` — abre/crea la base `nexora-offline`
- `put(store, item)` / `getAll(store)` / `delete(store, id)` / `clear(store)`
- Stores: `sync-queue` (operaciones pendientes), `dashboard-cache` (datos del dashboard)

### 2. `resources/js/lib/sync-queue.ts` (CREAR)
Cola de sincronización:
- `enqueue(operation)` — guarda operación con timestamp y metadatos
- `processQueue()` — intenta enviar todas las operaciones pendientes via fetch()
- `getPendingCount()` — cuántas operaciones están pendientes
- Cada operación: `{ id, type, endpoint, method, data, createdAt, retries }`
- Max retries: 5, backoff exponencial

### 3. `resources/js/Hooks/useNetworkStatus.ts` (CREAR)
Hook React:
- `isOnline` — boolean del estado actual
- `wasOffline` — true si recién se recuperó la conexión (para disparar sync)
- Escucha `online`/`offline` events del browser
- Cuando vuelve online → dispara `processQueue()`

### 4. `resources/js/Components/NetworkStatusBar.tsx` (CREAR)
Banner discreto en la parte superior:
- Offline: barra amarilla "Sin conexión — las operaciones se enviarán cuando vuelva internet"
- Syncing: barra azul "Sincronizando X operaciones pendientes..."
- Si hay operaciones encoladas, muestra badge con el count
- Se auto-oculta cuando todo está sincronizado

### 5. `app/Http/Controllers/Api/OfflineSyncController.php` (CREAR)
Endpoint API para recibir operaciones encoladas:
- `POST /api/v1/offline/sync` — recibe array de operaciones, las procesa en orden
- Retorna resultados: `{ results: [{ id, status, serverId?, error? }] }`
- Autenticación via token (Sanctum o Bearer simple)
- Rate limiting básico

### 6. `routes/api.php` (CREAR)
Rutas API para offline:
```php
Route::post('/offline/sync', [OfflineSyncController::class, 'sync']);
```

### 7. `public/sw.js` (MODIFICAR)
Agregar interceptación de requests POST/PUT cuando offline:
- Si el browser está offline y el request es POST/PUT → interceptar, guardar en IndexedDB via message al SW, retornar respuesta synthetic
- Cuando vuelva online → el SW notifica al client para procesar la cola

### 8. `resources/js/app.jsx` (MODIFICAR)
Inicializar:
- Registrar listener de sync queue
- Procesar cola al detectar que vuelve online
- Cachear datos del dashboard al cargar

### 9. `resources/js/Layouts/AuthenticatedLayout.jsx` (MODIFICAR)
Agregar `<NetworkStatusBar />` en el layout principal.

### 10. `resources/js/Pages/Sales/Pos/Index.jsx` (MODIFICAR)
Interceptar el submit del POS:
- Si online → enviar normalmente via Inertia
- Si offline → guardar en sync queue + mostrar toast "Venta guardada offline"

### 11. `resources/js/Pages/ServiceDesk/Ordenes/Create.jsx` (MODIFICAR)
Interceptar el submit de crear orden:
- Si online → enviar normalmente

## Flujo de sincronización

```
Usuario crea venta offline
  → sync-queue.ts encola en IndexedDB
  → Toast: "Venta guardada, se sincronizará cuando vuelva internet"
  
Internet vuelve
  → useNetworkStatus detecta 'online' event
  → NetworkStatusBar: "Sincronizando..."
  → sync-queue.processQueue() envía cada operación via fetch()
  → POST /api/v1/offline/sync con las operaciones
  → Backend procesa y retorna resultados
  → Si éxito: eliminar de la cola
  → Si error: reintentar (max 5 veces)
  → NetworkStatusBar se oculta
```
