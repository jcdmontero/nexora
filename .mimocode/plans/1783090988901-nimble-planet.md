# Fase 8: Real-time WebSocket — Plan de Implementación

## Alcance
- **Notificaciones push** en tiempo real (bell dropdown, count, toast)
- **Presencia** de usuarios (online/offline indicators)
- **Live updates** de estados de OT (cambios visibles al instante)
- **Chat interno** entre usuarios del mismo tenant
- **Servidor**: Soketi (Pusher-compatible) en Docker

---

## Fase 1: Infraestructura Broadcasting

### 1.1 Instalar dependencias

**Composer:**
```bash
composer require pusher/pusher-php-server
```

**NPM:**
```bash
npm install laravel-echo pusher-js
```

### 1.2 Configurar broadcasting

**Crear** `config/broadcasting.php`:
- Driver: `pusher` (Soketi es Pusher-compatible)
- App ID, key, secret, cluster from `.env`
- `use_tls => true` (Soketi default)
- `host` y `port` para Soketi (6001)

**Actualizar** `.env.example` y `.env`:
```
BROADCAST_CONNECTION=pusher
PUSHER_APP_ID=nexora
PUSHER_APP_KEY=nexora-key
PUSHER_APP_SECRET=nexora-secret
PUSHER_APP_CLUSTER=mt1
PUSHER_HOST=websocket
PUSHER_PORT=6001
PUSHER_SCHEME=https
VITE_PUSHER_APP_KEY="${PUSHER_APP_KEY}"
VITE_PUSHER_APP_CLUSTER="${PUSHER_APP_CLUSTER}"
VITE_PUSHER_HOST="${PUSHER_HOST}"
VITE_PUSHER_PORT="${PUSHER_PORT}"
VITE_PUSHER_SCHEME="${PUSHER_SCHEME}"
```

### 1.3 Docker: Agregar servicio Soketi

**Actualizar** `docker-compose.yml` — agregar servicio `websocket`:
```yaml
websocket:
  image: quay.io/soketi/soketi:latest
  container_name: nexora-websocket
  environment:
    SOKETI_DEFAULT_APP_ID: nexora
    SOKETI_DEFAULT_APP_KEY: nexora-key
    SOKETI_DEFAULT_APP_SECRET: nexora-secret
    SOKETI_DEFAULT_APP_ENABLE_CLIENT_MESSAGES: "true"
    SOKETI_PORT: 6001
  ports:
    - "6001:6001"
  networks:
    - nexora-network
  restart: unless-stopped
```

Actualizar `php-fpm` y `queue-worker` para que dependan de `websocket`.

### 1.4 Frontend: Echo setup

**Actualizar** `resources/js/app.jsx` — inicializar Echo después de `createInertiaApp`:
```js
import Echo from 'laravel-echo'
import Pusher from 'pusher-js'

window.Pusher = Pusher
window.Echo = new Echo({
  broadcaster: 'pusher',
  key: import.meta.env.VITE_PUSHER_APP_KEY,
  cluster: import.meta.env.VITE_PUSHER_APP_CLUSTER,
  wsHost: import.meta.env.VITE_PUSHER_HOST,
  wsPort: parseInt(import.meta.env.VITE_PUSHER_PORT || '6001'),
  wssPort: parseInt(import.meta.env.VITE_PUSHER_PORT || '6001'),
  forceTLS: (import.meta.env.VITE_PUSHER_SCHEME === 'https'),
  enabledTransports: ['ws', 'wss'],
})
```

### 1.5 Crear `routes/channels.php`

Definir canales de broadcast autorizados.

## Fase 3: Presencia + Live Updates

### 3.1 Presencia: Canal de presencia

**En** `routes/channels.php`:
```php
Broadcast::channel('presence.online', function ($user) {
    return [
        'id' => $user->id,
        'name' => $user->name,
        'email' => $user->email,
    ];
});
```

### 3.2 Presence Hook

**Crear** `resources/js/Hooks/usePresence.ts`:
- Se suscribe al canal `presence.online` con Echo.join()
- Retorna `{ onlineUsers: User[], isOnline: boolean }`
- Trackea join/leave events

### 3.3 Online indicators

**Modificar** sidebar o header para mostrar dots verdes en usuarios online (opcional, puede ser un componente discreto).

### 3.4 Live Order Updates Event

**Crear** `app/Events/OrdenEstadoActualizado.php`:
- Canal: `private tenant.{tenant_id}`
- Payload: `{ orden_id, numero_orden, estado_anterior, estado_nuevo, tecnico }`
- Se dispara en `OrdenController::cambiarEstado()` después de guardar

### 3.5 Frontend: useOrdenRealtime hook

**Crear** `resources/js/Hooks/useOrdenRealtime.ts`:
- Escucha canal `tenant.{id}` para evento `OrdenEstadoActualizado`
- Actualiza el estado local de la orden si el usuario está viendo esa OT
- Muestra toast de notificación
