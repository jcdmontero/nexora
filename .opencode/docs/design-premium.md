# DIRECTRIZ DE DISEÑO PREMIUM PARA TODA LA PLATAFORMA

## Objetivo

Toda la interfaz debe tener una apariencia Premium SaaS de nivel empresarial.

NO quiero una apariencia de plantilla genérica.
NO quiero una apariencia típica de Bootstrap.
NO quiero componentes visualmente básicos o anticuados.

El diseño debe transmitir:
- Profesionalismo
- Tecnología
- Innovación
- Confianza
- Escalabilidad
- Calidad empresarial

---

## Referencias Visuales

La calidad visual debe acercarse a la experiencia que ofrecen plataformas SaaS modernas como:
- Notion, Linear, Stripe Dashboard, Vercel, Clerk, Ramp, Mercury, Attio

No copiar diseños. Tomarlos únicamente como referencia de calidad, limpieza y consistencia.

---

## Filosofía de Diseño

Diseñar bajo estos principios: Minimalista, Moderna, Elegante, Profesional, Limpia, Premium, Consistente, Centrada en productividad.

**Evitar:** Diseños recargados, sombras excesivas, bordes gruesos, colores agresivos, componentes Bootstrap, apariencia de sistema antiguo.

---

## Componentes

### Botones
- Altura consistente, bordes suaves, espaciado interno generoso
- Hover/active/focus elegantes con transiciones fluidas
- Evitar: cuadrados, apariencia Bootstrap, gradientes exagerados

### Tarjetas (Card)
- Elemento visual principal del sistema
- Bordes sutiles, espaciado generoso, jerarquía clara, sombras discretas
- Cada tarjeta debe parecer un componente SaaS premium

### Formularios
- Inputs amplios, etiquetas claras, errores elegantes, focus moderno
- Evitar: inputs pequeños, formularios comprimidos, estilos Bootstrap

### Tablas
- Legacy empresarial moderna: buena legibilidad, espaciado, hover elegante
- Acciones visibles, paginación moderna, filtros integrados

### Dashboard
- KPIs destacados, gráficas profesionales, tarjetas informativas
- Jerarquía visual limpia y respirable. Evitar saturación.

---

## Sistema de Diseño

Todos los componentes deben seguir el mismo Design System. Consistencia en:
- Espaciados, tipografías, colores, bordes, sombras, iconografía, animaciones

No crear estilos aislados por pantalla.

---

## Calidad Esperada

Antes de dar por terminado cualquier componente, validar:
1. ¿Se vería profesional en un SaaS internacional?
2. ¿Parece una aplicación moderna de 2026?
3. ¿Tiene apariencia premium?
4. ¿Mantiene coherencia con el resto del sistema?

Si la respuesta es no → rediseñar.

> La prioridad no es terminar rápido. La prioridad es construir una experiencia visual premium, moderna, consistente y diferenciada.
